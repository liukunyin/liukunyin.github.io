Chapter 1

因特网?协议?

what’s the Internet?

Internet:network of network(网络套网络)
全球范围的网络,互联了全球设备的网络
ISP(服务提供商:本地,区域) 接入网络(无线,家庭,机构)

连接的计算机设备称为end system主机端系统,运行着网络应用
endsystem通过communication links(通信链路光纤无线电卫星…描述性能用传输速率bandwidth带宽)连接到网络
packet switch(分组(分组指数据块)交换设备(routers路由器,switchs分组交换机) 入接收,出转发)位于network core(网络核心)

端系统通过通信链路和分组交换机连接到一起
从发送端到接收端一个分组经历的一系列通信链路和分组交换设备成为path(路径)

常见终端接入设备
ADSL Modem(调制解调器),网卡,双角线,无线设备

端系统通过互联的ISP接入internet

protocols(协议)控制网络中信息的发送接收
api(application programming interface)

what’s a protocol?

协议定义了在两个或多个通信实体之间交换的报文格式和顺序以及报文发送接收或有其他事件产生时采取的一系列动作

network edge;

网络边缘包括hosts(主机), access net(接入网络), physical media(链路物理媒介)
主机分为客户机和服务器

接入网将终端系统连接到边缘路由器(端系统到其他远程端系统的第一台路由器),

三种接入网络
家庭接入: dsl(数字用户线缆) 电缆 fftth 拨号 卫星
机构接入: 以太网 wifi
移动无线接入: 4g lte
带宽?共享还是专用?

network core:(相互连接的路由器和网络的网络)

packet/circuit switching(分组/电路(链路)交换)
电话网络电路交换
Internet分组交换

packet switching

store-and-forward transmission(存储转发机制)
必须接收完全部数据才可以传输到下一个链路,每一个节点必须先存储后转发
queueing delay(排队时延)
每个链路有输出缓存(存储路由器要发往下一链路的分组),到达的分组要等待之前正在传输的分组
loss(丢包)
因为缓存空间有限,空间已满时会出现丢包现象

网络核心的两个关键功能routing,forwarding(路由,转发)
路由:决定分组由源到目的地的路径选择
转发:将数据包有路由器的输入移动到输出的过程

circuit switching

FDM频分多路复用
TDM时分多路复用

Internet structure

终端系统通过ISP接入Internet

描述网络性能loss(丢包), delay(时延), throughput(吞吐量)

四种delay(总时延=4种时延之和)
processing:节点处理时延

  1. 校验bit位错误
  2. 确定输出链接
  3. 小于ms

Queueing(等待传输):排队时延
Transmission:传输时延
Propagation:传播时延

throughput:输入方到输出方传输bit的速率

security

protocol layers, service models

五层
application应用层:支持网络应用(HTTP,SMTP,FTP)
应用层协议分布在多个端系统上,一个端系统中的应用程序使用协议与另一个端系统中的应用交换信息的分组(packets of information).我们把这种位于应用层的信息分组成为message(报文)

transport传输层:实现进程到进程的数据传输(TCP UDP)
transport-layer packet(运输层分组)称为segment(报文段)

network网络层:实现从源到目的地的datagram数据报路由(IP)
网络层的分组成为datagram(数据报)

link链路层:在相邻网络元素之间进行数据传输(以太网 wifi ppp)
链路层分组称为frame(帧)
physical物理层:实现bit流

OSI model(开放式系统互联)

session会话层
提供了数据交换定界和同步功能(包括了建立检查点和恢复方案的方法)
